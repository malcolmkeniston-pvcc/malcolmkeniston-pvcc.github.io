Chapter 5: Programming Algorithms and Software Testing
Phases of computer development
Phase 1: Understand the problem
Phase 2: Design an algorithm to solve the problem
Phase 3: Write the program code in the appropriate language based on the algorithm
Phase 4: Enter the program and save
Phase 5: Execute the program
Phase 6: Test the program, find errors and debug, then retest
Phase 7: Document and evaluate the program

Programming structures:

  • Sequential structure: Top-down execution
  • Decision structure: if/then/else
  • Iterative (Looping)
    • while loop: continue looping as long something is true repeat/until loop: continue looping until something is true for loop: loop a given number of times
  • Recursive structures:
    • Repeat a set of instructions as a subtask of itself Repetition continues until: degenerative case, which is a termination condition, is met (degenerative case is also called a "base case")

Two classic programming/database problems

  • Sorting: Descending order/Ascending order
  • Sorting numbers
  • Sorting strings:(in accordance with ASCII values)
  • Searching:find specific data that meet a certain criteria
  • In database:
  • Filters:temporary search
  • Queries:search criteria are saved so they don't have to be recreated
  • Two popular search techniques
    • Squential search: search from top to bottom or bottom to top, one item at a time
    • Binary search
    • only works if data is SORTED first
    • start in the middle, then go to half of that, etc.
  • Two classic program goals:
    • Efficiency: Program takes minimal time and/or resources
    • Correctness:Program runs in expected ways and works for all expected conditions

    Software testing and verification

    Kinds of test:

    • Acceptance testing: Testing to see if the software meets Guidelines
    • Intergation testing: Testing to see if the software works
    • Unit testing: Testing to see if the software works on a unit
    • Functional testing: Testing to see if the software works on a function level
    • Performance testing: Testing to see if the software works on a performance level
    • Regression testing: Testing to see if the software works on a regression level
    • Stress testing: Testing to see if the software works on a stress level
    • Usability testing: Testing to see if the software works on a usability level

    Software testing is done to detect:

    • Architectural flaws
    • Poor design decisions
    • Invalid or incorrect funtionality
    • Security vulnerabilities
    • Scalability issues

    Testing best practices

    • Continous testing: Team tests out each build until it is available
    • Configuration management: Maintain test assets and track software build
    • Service virtualization: Simulates the service and system that are missing or not completed
    • Defect or "bug tracking": Monitoring defects to improve qaulity
    • Metrics and reporting: Report status, goals, and results by team
