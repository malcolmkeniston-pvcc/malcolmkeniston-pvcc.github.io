  1. Q: What is a version control system?
    • A: Version control systems are software tools that help software teams manage changes to source code over time.
  2. Q: What is Git?
    • A: Git is a version control system used for tracking changes in comptuer files.
  3. Q: What is GitHub?
    • A: GitHub is a code hosting platform for version control and collaboration.
  4. Q: What is the difference between Git and Github?
    • A: Git is a version control system that allows developers to track changes in their code. GitHub is a web-based hosting service for git repositories.
  5. Q: What company owns it now?
    • Microsoft.
  6. Q: How much does a Github account cost?
    • A: Free for standard users, but there is different payment plans for teams and enterprises
  7. Q: What is the Octocat?
    • A: The mascot of the hosting service GitHub.

      Git/GitHub Terms

    • repository: a place where you can store your code, files, and each file's revision history
    • commit: records changes to one or more files in your branch.
    • fork: a new repository that shares code and visibility settings with the original "upstream" repository
    • push: how you transfer commits from your local repository to a remote repo
    • pull requests: lets you tell others about changes you've pushed to a branch in a repository on GitHub
    • workflows: configurable automated process that will run one or more jobs
    • issues: lets you track your work on GitHub, where development happens.
    • raw button: you can view or copy the raw content of a file without any styling
    • blame button: used to examing the contents of a file line by line and see when each line was last modified and who the author of the modifications was
